# 研究资料索引

## 1. 标准组织 / 行业组织官网

这是最值得先看的地方，因为你要研究的是"体系"，不是几个公司故事。

**优先看：**

- [OpenADR Alliance](https://openadr.org/)
- [LF Energy](https://lfenergy.org/)
- [SEPA（Smart Electric Power Alliance）](https://www.sepapower.org/)
- [IEEE 标准介绍页](https://standards.ieee.org/)
- [IEC 标准介绍页](https://www.iec.ch/)

你要收集的不是技术细节本身，而是：

- 他们解决什么问题
- 为什么需要开放标准
- 有哪些参与方
- 实际落地在哪些场景

**输出物：** 做一张表：

| 组织/标准 | 解决的问题 | 面向谁 | 开放到什么程度 | 落地难点 |
| --------- | ---------- | ------ | -------------- | -------- |

## 2. 龙头公司的白皮书 / 技术博客 / 开放战略

选几个典型样本：

- [丰田（精益生产方法论扩散）](https://global.toyota/en/)
- [特斯拉（开放专利 / NACS）](https://www.tesla.com/)
- 华为（[openEuler](https://www.openeuler.org/zh/) / [OpenHarmony](https://www.openharmony.cn/)）
- [宁德时代（如果有开放平台/标准参与信息）](https://www.catl.com/)
- [Schneider](https://www.se.com/) / [Siemens](https://www.siemens.com/) / [ABB（工业与能源巨头）](https://new.abb.com/)
- Google / Linux / Kubernetes（基础设施开源作为参照组）

你要问：

- 它开放了什么
- 它没开放什么
- 它为什么开放
- 开放之后得到什么，失去什么

**输出物：** 做"案例卡片"，每家公司一页。

## 3. GitHub / 基金会项目页面

因为你研究的是开源，不看真实社区就会变成纯概念。

**重点看：**

- 项目 README 怎么定义问题
- governance 怎么设计
- 谁在贡献
- roadmap 指向哪里
- issue 里真实痛点是什么

**重点项目可先看：**

- LF Energy 下面的项目
- OpenADR 相关 SDK / reference implementation
- OpenEMS
- openleadr
- Eclipse Foundation 工业/能源相关项目

**输出物：** 记录：

- 项目目标
- 参与者结构
- 技术切口
- 社区活跃度
- 它到底是在做"工具"，还是"生态基础设施"

## 4. 中国语境材料

这一块很重要，因为你不是只做国际技术综述，你还想结合中国工业与毛泽东思想分析。

**去看：**

- [工信部](https://www.miit.gov.cn/)、[能源局](https://www.nea.gov.cn/) 的政策文件
- [中国标准化协会](https://www.cnsa.org.cn/) 的政策文件
- [中国发改委](https://www.ndrc.gov.cn/) 的政策文件
- [中国信通院](https://www.caict.ac.cn/)
- 新型电力系统、工业互联网、双碳相关政策
- "卡脖子""自主可控""标准体系建设"相关表述
- 国内企业开源战略文章

**你要抓的是：**

- 中国工业/能源体系最现实的矛盾是什么
- 为什么"自主可控"会和"开放协作"同时存在
- 哪些领域政策支持标准开放，哪些领域更强调安全和控制权
