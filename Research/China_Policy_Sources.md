# 中国政策材料索引（工业开源相关）

> 说明：本页先做“材料底座”，聚焦与工业开源研究高度相关的中国政策与研究来源。当前阶段重点不是做完整政策综述，而是建立后续可持续扩展的索引：哪些文件值得读、为什么值得读、它们对 OpenIndustry 的研究有什么启发。

## 使用口径

这里不把“开源”狭义理解为 GitHub 代码发布，而是从更接近中国政策语境的几个关键词切入：

- 标准体系
- 平台协同
- 互联互通
- 数字化底座
- 源网荷储协同
- 自主可控
- 数据与软件基础设施

这些表述虽然未必直接使用“开源”一词，但与工业开源研究中的“开放接口、公共底座、参考实现、跨主体协作、技术控制权”高度相关。

## 初步观察

### 1. 中国政策很少直接说“开源”，但大量在谈“协同”和“标准”
从目前已经纳入的材料看，中国官方文件更常使用的词是：
- 标准体系
- 协同互动
- 智慧化调度
- 平台建设
- 数字化智能化
- 数据贯通
- 自主可控

这意味着，如果要在中国语境里研究工业开源，不能只搜“开源”二字，而必须把“开源”翻译成更符合政策语言的结构性问题：
- 接口是否开放
- 标准是否统一
- 平台是否可互联
- 工具链是否可共享
- 关键能力是否掌握在自己手里

### 2. 能源侧政策材料比工业侧更容易直接连接“开放协同”
能源体系当前已经明显出现：
- 源网荷储协同
- 虚拟电厂
- 智能微电网
- 需求侧响应
- 智慧调度
- 配电网标准修订

这些都天然要求多主体互联和跨系统协作，因此特别适合与开放标准、开源参考实现、共享数字基础设施等主题对接。

### 3. 工业侧最关键的两条线已经开始浮现：标准体系 + 开源生态
随着工信部《工业互联网综合标准化体系建设指南》和信通院《开源生态白皮书》纳入索引，这份材料已经不再只是能源数字化侧的资料表，而开始具备“工业开源”研究的基本骨架：
- 工信部文件定义工业互联、标准体系、平台协同的制度底座
- 信通院文件补充开源生态、社区、企业采用和产业协作的现实基础

这两条线后续如果继续补强，OpenIndustry 的“中国语境”部分就会更扎实。

### 4. 后续仍需继续补工信部工业软件和信通院工业互联网平台研究
虽然这次已经补进了关键文件，但仍然建议下一步继续扩展：
- 工信部关于工业软件、智能制造、关键基础软件、自主可控的正式文件
- 信通院关于工业互联网平台、工业数字化转型、基础软件能力的研究报告

只有补齐这部分，OpenIndustry 才能把“工业开源”与“中国制造业数字化”真正稳定地连接起来。
