# 中国政策材料索引（工业开源相关）

> 说明：本页先做“材料底座”，聚焦与工业开源研究高度相关的中国政策与研究来源。当前阶段重点不是做完整政策综述，而是建立后续可持续扩展的索引：哪些文件值得读、为什么值得读、它们对 OpenIndustry 的研究有什么启发。

## 使用口径

这里不把“开源”狭义理解为 GitHub 代码发布，而是从更接近中国政策语境的几个关键词切入：

- 标准体系
- 平台协同
- 互联互通
- 数字化底座
- 源网荷储协同
- 自主可控
- 数据与软件基础设施

这些表述虽然未必直接使用“开源”一词，但与工业开源研究中的“开放接口、公共底座、参考实现、跨主体协作、技术控制权”高度相关。

## 初步观察

### 1. 中国政策很少直接说“开源”，但大量在谈“协同”和“标准”
从目前已经纳入的材料看，中国官方文件更常使用的词是：
- 标准体系
- 协同互动
- 智慧化调度
- 平台建设
- 数字化智能化
- 数据贯通
- 自主可控

这意味着，如果要在中国语境里研究工业开源，不能只搜“开源”二字，而必须把“开源”翻译成更符合政策语言的结构性问题：
- 接口是否开放
- 标准是否统一
- 平台是否可互联
- 工具链是否可共享
- 关键能力是否掌握在自己手里

### 2. 能源侧政策材料比工业侧更容易直接连接“开放协同”
能源体系当前已经明显出现：
- 源网荷储协同
- 虚拟电厂
- 智能微电网
- 需求侧响应
- 智慧调度
- 配电网标准修订

这些都天然要求多主体互联和跨系统协作，因此特别适合与开放标准、开源参考实现、共享数字基础设施等主题对接。

### 3. 工业侧的材料链正在从“标准体系”走向“企业实施路径”
随着《工业互联网综合标准化体系建设指南》《“十四五”智能制造发展规划》《国家智能制造标准体系建设指南》《制造业企业数字化转型实施指南》《关于加快传统制造业转型升级的指导意见》被纳入，工业侧材料已经不只是讲原则，而开始出现“企业怎么落地、如何改造、如何升级”的路径。

这很重要，因为 OpenIndustry 研究的不应只是制度和理念，还应看到：开放标准、工业软件、平台能力和数字化改造，最终必须进入企业实施过程。

### 4. 当前最值得继续补的是“工业软件/基础软件/自主可控”的更直接政策文本
虽然工业材料链已经更完整，但还缺更尖锐的一类文件：
- 直接讨论工业软件、关键基础软件、软件更新替代、自主可控能力建设的政策材料

如果补上这类文件，OpenIndustry 对“中国工业开源”的研究会更容易回答一个核心问题：

**为什么中国制造业不仅需要标准和平台，也需要更强的可掌握软件底座。**

## 后续扩展建议

后面可以沿着三条线继续扩展：

1. **工信系统线**：工业互联网、智能制造、工业软件、标准体系、自主可控  
2. **能源系统线**：新型电力系统、能源数字化、虚拟电厂、需求响应、配电网标准  
3. **产业研究线**：信通院、研究院、行业白皮书，对政策目标与产业落地之间的距离做补充判断
