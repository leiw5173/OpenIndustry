# Contributing to OpenIndustry

Thank you for your interest in contributing. This document explains the rules and process for participation.

---

## Code of Conduct

All contributors are expected to:

- Be respectful and constructive in all discussions
- Focus on technical merit and the project's goals
- Welcome contributors of all backgrounds and experience levels
- Avoid personal attacks, harassment, or discriminatory language

Violations may result in removal from the project.

---

## Contribution Rules

### 1. License Agreement

By submitting any contribution (code, documentation, data, or other materials), you agree that your contribution is licensed under **GPL-3.0**. Do not submit code that you do not have the right to license under GPL-3.0.

### 2. One Issue, One Pull Request

- Each pull request should address a single concern
- Link your PR to the relevant issue (`Closes #123`)
- Do not bundle unrelated changes in one PR

### 3. Discuss Before Building

For significant changes (new features, architectural decisions, research directions):

1. Open an issue first to describe what you want to do and why
2. Wait for feedback from maintainers before investing significant effort
3. Small fixes (typos, broken links, minor clarifications) can go straight to a PR

### 4. Research Contributions

Since this project is in its research phase, contributions to `RESEARCH.md` and related documents are especially welcome. When contributing research:

- Cite your sources
- Distinguish between established facts and your own analysis
- Flag areas of uncertainty clearly

### 5. Commit Messages

Use clear, descriptive commit messages:

```
docs: expand case study on OpenADR ecosystem
research: add landscape survey for LF Energy projects
fix: correct broken link in README
```

Prefix types: `docs`, `research`, `feat`, `fix`, `refactor`, `test`, `chore`.

### 6. Language

- Code, comments, commit messages, and issue titles must be in **English**
- Discussion in issues and PRs may be in English or Chinese
- Documentation should be in English (Chinese translations welcome as separate files)

---

## How to Contribute

1. Fork the repository
2. Create a branch: `git checkout -b your-topic`
3. Make your changes
4. Open a pull request against `main`

---

## Maintainers

Maintainers review PRs, triage issues, and make decisions when consensus is unclear. The current maintainer list is in [GOVERNANCE.md](GOVERNANCE.md).

---

## Questions?

Open an issue with the `question` label.
